GENERAL STEPS

1. SPIDERING
  • Map manually every link, form, functions (with javascript and cookies enabled/disabled)
  • Explore manually generated site map and visit all non-visited paths

2. DISCOVER HIDDEN CONTENT
  • Use Intruder and Discover content (will automate ALL of hidden content discovery methods)
  • Brute force dirs http://targetdomain.com/<dir>/(set position such: GET /§§/ HTTP/1.1)
  • Brute force deeper paths ie. http://targetdomain.com/auth/<file>(set position such: GET /auth/§§ HTTP/1.1)
  • Infer naming schemas from site map
  • Check all HTML files for information disclosure in comments (Engagement tools > Find Comments - sort by comment)
  • Compare cache snapshots (see Recon > Passive > Cache Search)

3. BRUTE FORCE OTHER RELEVANT PREDICTABLE PATHS BASED ON USED WEB TECHNOLOGIES

4. BRUTE FORCE DIRECTORIES WHICH RETURNED ACCESS DENIED OR UNAUTHORIZED

5. GREP ACCESS DENIED OR UNAUTHORIZED DIRS WITH waybackurls OUTPUT AND FORCE BROWSE

6. DISCOVER FUNCTIONAL PATHS
  • Identify pages which are served by passing function name as parameter to request (ie. /admin.jsp?action=editUser)
  • Infer function names

7. DISCOVER HIDDEN PARAMETERS


8. DISCOVER DIFFERENT VERSION OF APP FOR DIFFERENT USER-AGENTS
  • Send app root to intruder and attack user-agent with user-agents burp list

9. DISCOVER OUT-OF-BAND CHANNELS
  • Inspect the source code for
    • API channels
    • WebSocket channels

10. DISCOVER COMMON LOGIN PAGES

11. DISCOVER COMMON PASSWORD FILES

12. DISCOVER EXISTING BACKDOORS

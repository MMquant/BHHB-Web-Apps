Logic data validation

Test if malicious data can be injected into application so that app will consider them as validated/secure.
For example in multistage process examine if some parameters are passed in more than in one process. If so,
try to tamper parameters in later stage.
Application/system must ensure that only “logically valid” data is accepted at all input and hand off points of the applica- tion or system and data is not simply trusted once it has entered the system.

Test Ability to forge requests

Test if pages which shouldn't be reached given specific time or circumstances can be reached.

Test for Process Timing

Test if any data or application behaviour can be deduced by request timing measure

Test Number of Times a Function Can be Used Limits

Verify that the application does not allow users to exercise its functions more times than required by the business logic workflow.

Testing for the Circumvention of Work Flows

Test for skipping, repeating or mixing stages in multistep process

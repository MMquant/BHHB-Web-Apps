Identify session management parameters
  • Session can be managed by cookies, hidden form fields, URL parameters
  • Create cookies list (Content Discovery -> Search "Set-cookie")
    • Page which sets cookie
    • cookie name
    • cookie directives
    • cookie purpose
  • Observe which items are passed to browser after authentication
  • If not obvious, login as valid user and remove selected parameters one-by-one and check which request returns non-authenticated page
  • Inspect if any cookies get modified by any pages. What events modify cookie?
  • Which pages needs this cookie to be working properly

Check for HTTP stateless session management
  • Check request headers for credentials
  • How to identify stateless session management
    • Token-like data items issued to the client are fairly long (100 or more bytes)
    • The application issues a new token-like item in response to every request
    • The data in the item appears to be encrypted (and therefore has no discernible structure) or signed (and therefore has a meaningful structure accompanied by a few bytes of meaningless binary data)
    • The application may reject attempts to submit the same item with more than one request

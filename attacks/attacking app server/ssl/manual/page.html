Every service wrapped into SSL must be tested

Check
  • sensitive information transmitted over HTTP (Credentials in URL, POST body, Basic auth headers)
  • Secure flag for Session Cookies
  • Webserver response headers should contain Strict-Transport-Security header. Otherwise SSL communication can be stripped.
  • presence of HTTP and HTTPS both, which can be used to intercept traffic
  • presence of mixed HTTPS and HTTP content in the same page, which can be used to leak information

Defects in multi-stage login
  • Perform valid login and record all inputs to the web app
  • Identify all inputs in all login stages and discover if any inputs are resubmitted (in parameter, cookie, hidden field, ...)
  • Repeat login more times
    • Login steps in different sequence
    • Proceed directly to any given stage and continue from there
    • Skip each stage and proceed with next
  • Change resubmitted inputs at different stages to see if login will be still successful - if yes, inputs may be validated just in first stage and can be changed arbitrarily in subsequent stages
  • Try to switch users in different stages (1st stage: provide user1:password, 2nd stage user2:pin)
  • Inspect data which are given back in response, it may contain login information which can be modified

Security questions
  • Examine all pre-generated questions. Are answers predictable?
  • Self-generated questions are always insecure.
  • If there is "question" verification at some stage inspect if question is returned back to server via hidden form
  • Even if questions are changing we can login many times and wait for proper question every time
  • Can users enumerated with "questions" functionality?

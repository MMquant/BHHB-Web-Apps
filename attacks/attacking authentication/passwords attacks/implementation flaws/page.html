Fail-open login mechanism
  • Perform valid login and record all inputs to the web app
  • Login again and modify inputs
    • a. Submit an empty string as the value.
    • b. Remove the name/value pair altogether.
    • c. Submit very long and very short values.
    • d. Submit strings instead of numbers and vice versa.
    • e. Submit the same item multiple times, with the same and different values.
  • Review responses with comparer
  • Combine inputs which changed response and web app behaviour
  • If there is multistep login note all input params and cross-request them in different stages

Insecure Storage of Credentials
  • Is password in server responses?
  • If we can retrieve passwords
    • check for non-unique hashes (common passwords or non-salted passwords)
    • query online rainbow tables for unsalted hashes

Weak lock-out mechanism
  • Try incorrectly login 3x. Can you log in with correct credentials on 4th trial?
  • Try incorrectly login 4x. Can you log in with correct credentials on 5th trial? ...
  • Determine how many incorrect inputs needed to lockout.
  • Attempt to login 5 min later, 10 min later ...
  • Determine when is account unlocked
  • Test unlock mechanism (unlock URL, security questions)

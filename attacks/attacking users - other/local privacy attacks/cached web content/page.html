Testing for Browser cache weakness
  • Clean cookies and cache in browser
  • Login in and get some sensitive data
  • Review files generated in browser cache (use search fro specific string)
  • Inspect server responses where are sensitive data. Such responses must instruct browser to not cache these responses via headers
    • Cache-Control: no-store, no-cache, must-revalidate, post-check=0, pre-check=0
    • Pragma: no-cache
    • Expires: 0
  • Inspect <meta> tag for these headers
  • If browser store sensitive data in cache they can be found
    • Mozilla Firefox:
      • Unix/Linux: ~/.mozilla/firefox/<profile-id>/Cache/
      • Windows: C:\Documents and Settings\<user_name>\Local Settings\Application Data\Mozilla\Firefox\Profiles\<profile-id>\Cache
    • Internet Explorer:
      • C:\Documents and Settings\<user_name>\Local Settings\ Temporary Internet Files\Content.IE5

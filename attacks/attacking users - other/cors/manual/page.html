CORS Headers

All non-GET and non-POST methods must be precommunicated by pre-flight OPTION request with proper headers

Origin & Access-Control-Allow-Origin

Originis request header
Access-Control-Allow-Origin is response header and contains domains which are able to read this response

Access-Control-Request-Method & Access-Control-Allow-Method

Access-Control-Request-Method is request header in preflight OPTIONS request
Access-Control-Allow-Method is response header which controls which methods is client allowed to request

Access-Control-Allow-Credentials

This header as part of a preflight request indicates that the final request can include user credentials


Check
  • Access-Control-Allow-Origin shouldn't be set to '*'
